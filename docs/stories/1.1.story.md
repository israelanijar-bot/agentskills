# Story 1.1 — Landing Page e Fluxo de Compra do OpenClaw PT Pack

## Status: Approved

## Story Statement

**Como** visitante do AgentSkills.com.br,
**Eu quero** ver uma landing page dedicada do "OpenClaw PT Pack" com descrição, preço e botão de compra,
**Para que** eu possa entender o produto, comprá-lo via Stripe e fazer o download dos arquivos após a compra.

## Escopo

Implementar o fluxo completo end-to-end do primeiro produto digital do marketplace:

1. Landing page dedicada em `/skills/pack-openclaw-pt`
2. Produto registrado no seed data
3. Checkout via Stripe (infraestrutura existente)
4. Webhook Stripe grava purchase no Supabase
5. Download autenticado via Supabase Storage signed URLs
6. Página "Minhas Compras" mostra compras reais com botão de download
7. Página de sucesso pós-compra com botão de download
8. Criação dos arquivos do pack (SOUL.md, AGENTS.md, BOOT.md, HEARTBEAT.md, MEMORY.md, IDENTITY.md) + guia de setup

## Acceptance Criteria

- [ ] **AC1**: Landing page `/skills/pack-openclaw-pt` renderiza com título, descrição, preço (R$ 47,00), lista de arquivos incluídos e botão "Comprar Agora"
- [ ] **AC2**: Botão "Comprar Agora" redireciona para Stripe Checkout com o produto correto
- [ ] **AC3**: Webhook Stripe (`/api/webhook/stripe`) grava registro na tabela `purchases` do Supabase após pagamento confirmado
- [ ] **AC4**: Migration SQL cria tabela `purchases` com campos: id, user_id, product_slug, stripe_session_id, stripe_payment_intent, amount, currency, status, created_at
- [ ] **AC5**: Rota API `/api/download/[slug]` valida autenticação, verifica purchase no Supabase e retorna signed URL do Supabase Storage (expira em 60min)
- [ ] **AC6**: Página `/compra/sucesso` exibe confirmação e botão "Baixar Arquivos" que chama a API de download
- [ ] **AC7**: Página `/minhas-compras` lista compras reais do Supabase com botão de download para cada produto comprado
- [ ] **AC8**: Componente `DownloadButton` exibe estados: loading, download ready, erro
- [ ] **AC9**: Seed data em `seed.ts` contém o produto "OpenClaw PT Pack" com todos os campos (slug: `pack-openclaw-pt`, type: `bundle`, price: 4700)

- [ ] **AC11**: `npm run build` passa sem erros

## Decisões de Arquitetura (aprovadas pelo Axel)

| ID | Decisão | Justificativa |
|----|---------|---------------|
| D1 | Rota dedicada `/skills/pack-openclaw-pt` | Landing page otimizada, independente do template genérico de produto |
| D2 | Produto no seed.ts (não migração Supabase) | Consistente com padrão atual do projeto; migração de dados é escopo separado |

## Dev Notes

### Fluxo Técnico Completo

```
Visitante → /skills/pack-openclaw-pt (landing)
  → Clica "Comprar Agora"
  → /api/checkout (cria Stripe Session)
  → Stripe Checkout (pagamento)
  → Stripe webhook → /api/webhook/stripe
    → Grava purchase no Supabase
  → Redirect → /compra/sucesso?session_id=X
    → Exibe botão "Baixar Arquivos"
    → Clica → /api/download/pack-openclaw-pt
      → Verifica auth + purchase
      → Gera signed URL do Supabase Storage
      → Redireciona para download
```

### Infraestrutura Existente (reutilizar)

- `/api/checkout/route.ts` — já cria Stripe Checkout Session
- `/api/webhook/stripe/route.ts` — existe mas precisa completar lógica de purchase
- `src/lib/stripe.ts` — cliente Stripe configurado
- `src/lib/supabase/` — clientes server/client configurados
- `src/middleware.ts` — proteção de rotas autenticadas
